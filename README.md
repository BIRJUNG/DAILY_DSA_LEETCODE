## 📌 About This Repository

This repository is my **daily DSA and LeetCode problem-solving journal**.

The goal of this repo is not just to store solutions, but to document:

- how I approached each problem
- what my first thought process was
- different ways to solve the same question
- time and space complexity tradeoffs
- edge cases
- interview takeaways
- what I learned from each problem

Instead of uploading only code, I am maintaining each problem as a **well-explained markdown note**, so this repository becomes both:

- a **revision resource for interviews**
- a **proof of problem-solving ability**
- a **record of consistency and growth**

---

## 🎯 Why I Built This Repo

I wanted this repository to reflect **real understanding**, not copy-paste answers.

For each problem, I try to capture:

- my own intuition
- the brute-force or first natural idea
- the improved or optimized approach
- why the final solution works
- where I got confused and what I learned

This makes the repo more meaningful than a collection of accepted submissions.

## 🚀 My Approach to Problem Solving

My focus is not only on solving the question, but on understanding:

### 1. The natural first idea

What comes to mind first when I see the problem?

### 2. The brute-force baseline

Even if it is not efficient, can I first think of a correct solution?

### 3. The optimization step

How do I improve the approach?

### 4. The pattern

What interview pattern does the problem belong to?

### 5. The takeaway

What should I remember when I see a similar question again?

## 🧩 Sample Problem Workflow

For each LeetCode problem, I usually follow this process:

1. Read the question carefully
2. Think of the most natural approach
3. Solve it in the simplest correct way
4. Improve it if possible
5. Write the explanation in my own words
6. Add complexities, edge cases, and takeaways
7. Commit it as part of my daily practice
